Exploring the guide on Quarto.org

This guide on quarto.org is great tool to start this website. However, it took me some time to get familiarized with it. I started by running the codes for the examples for what I liked and made changes to them. I was also doing Google searches of what I wanted to do with my website before realizing that most of the information I needed was in this guide!

This was before I realized that the Prepare for Class has a great reference for what I needed. This How to Build a Quarto blog would have been really helpful at the beginning of the assignment.

Copying code blocks

The little button that on the top right corner of the code blocks serves as a great tool to copy and paste the code blocks. I didn’t understand why turning off echo or applying code folding didn’t work properly. I spent a good hour trying to find different ways to do the same thing. The problem was that I was highlighting the codes and copying them instead of using this button!

To see an example, expand the note below.

Note

To show the copy button for your own website, add this in your title block:

format:
  html:
    code-copy: true

Valid values for code-copy include:

hover Show button on hover (default)
true Always show code copy button
false Never show code copy button

Understanding how to upload Quarto files to GitHub

Before publishing the changes to my website, I had to keep reminding myself of the following:

  • Manually set the directory to belkhuu.github.io, just in case I was working on something else.

  • Render the website on RStudio before uploading file from the _site folder to GitHub

  • Only pull what’s inside the _site folder to GitHub and not the folder itself
